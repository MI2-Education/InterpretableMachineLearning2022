California Housing price regression and Ceteris Paribus¶

The report describes XGBoost and Multi Layer Perceptron learning results on California Housing dataset and decomposition of trained models sample predictions using Ceteris Paribus. The dataset contains 8 features describing blocks of houses. MedHouseVal is an explained variable containing medium house value in $100 000. Below there are quartiles for all of the features. Some features like "average number of rooms" may take very large values because the dataset contains also vacation resorts data.

Below there is a decomposition of selected predictions using Ceteris Paribus for house age, average number of bedrooms, latitude and longitude. The first four figures show median house value for changes in one feature for observation with lower price. As we can see, house age does not influence the price too much. Only age lower than 2 years would increse the price of houses in the block. The plot with average number of bedrooms makes sense only for less than 5, because average number of rooms is less than 5. If there were more or less bedrooms than in reality, the price would be higher. Latitude and longitude have the largest impact on price for this observation and change in latitude could increse the price even more than 10 times.

The next four figures show decomposition for the higher price. Similarily to the previous case, changes in coordinates would influence the price the most. The largest difference there is for the house age feature. If there was similar house in the same location, the same number of rooms and other similar features which has been built more recently, then its price would be lower. It is quite surprising because newer houses should be more expensive. Possible hypothesis is that newer houses built in this place are less equipped.

Below there are results of a similar Ceteris Paribus decomposition for the lower and higher price (in this order), but using MLP model. As we can see, changes in location would also influence the price significantly for this model. However, there is a difference for the average number of bedrooms, especially for the observation with higher price. The price was nearly constant for any number of bedrooms more than 2 for the previous model. In the case with MLP model and the observation with higher price, for more than 2 bedrooms, the more bedrooms, the lower prediction result. Possible hypothesis is that higher difference between number of rooms and number of bedrooms could be more important than higher raw number of bedrooms. The higher the difference, the more guest and living rooms there are in a house.

from matplotlib import pyplot
import dalex as dx
from xgboost import XGBRegressor

from sklearn import metrics
from sklearn.model_selection import train_test_split
from sklearn.datasets import fetch_california_housing
from sklearn.preprocessing import MinMaxScaler
from sklearn.pipeline import make_pipeline
from sklearn.neural_network import MLPRegressor

def evaluate(testY, predY):
    pyplot.plot(testY, testY, 'r')
    pyplot.scatter(testY, predY)
    pyplot.title('Median House Value');
    pyplot.xlabel('Actual')
    pyplot.ylabel('Predicted')
    pyplot.show()

    print()
    print("MSE:", metrics.mean_squared_error(testY, predY))
    print("MAE:", metrics.mean_absolute_error(testY, predY))
    print("R2:", metrics.r2_score(testY, predY))
